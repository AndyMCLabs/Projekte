Project-Checkliste

Diese Seite hilft dir, dein Projekt zu perfektionieren!

1. HTML - Webseite erstellen

Was ist wichtig?

  • HTML Grundgerüst einbinden.
  • HTML Sprache auf "de" ändern.
  • HTML Titel ändern.
  • Erforderliche Metadaten einfügen.
  • Überprüfen, ob Semantik passt.
  • Jedes Element hat eine ID oder Klasse.
  • Header und Footer eingefügt.
  • ARIA-Labels eingefügt.
  • Verwendung von <main> für den Hauptinhalt.
  • Links auf andere Seiten oder Ressourcen richtig implementiert.
  • Alt-Attribute für Bilder hinzugefügt.
  • Formularfelder (wenn vorhanden) mit <label> versehen.
  • Dokument validieren (z.B. mit dem W3C Validator).
  • Konsistente Einrückung verwenden.
  • Semantische HTML-Elemente verwenden.
  • Eindeutige und beschreibende IDs und Klassen verwenden.
  • Inline-Stile vermeiden.
  • HTML-Entitäten für Sonderzeichen verwenden.
  • Unnötige Tags vermeiden.
  • Zugänglichkeit berücksichtigen.
  • Kommentare für Klarheit verwenden.
  • Korrekte Dokumentstruktur sicherstellen.
  • Verwendung von <meta>-Tags sicherstellen.

2. CSS - Stylesheet erstellen

Was ist wichtig?

  • Externe CSS-Datei verwenden.
  • Selbsterklärende und konsistente Klassennamen verwenden.
  • Verwendung von semantischen Elementen fördern.
  • Verwendung von BEM (Block-Element-Modifier) zur Strukturierung der Klassen.
  • Konsistente Einrückung und Formatierung der CSS-Regeln.
  • Verwendung von CSS-Variablen zur Vereinfachung von Wiederholungen.
  • Minimale und übersichtliche Selektoren verwenden.
  • Box-Modell-Eigenschaften (Padding, Margin) korrekt verwenden.
  • Media Queries für responsive Designs nutzen.
  • Vermeidung von Inline-Stilen.
  • Konsistente Farbpalette und Schriftarten verwenden.
  • Verwendung von Kommentaren zur Erklärung komplexer Styles.
  • Animations- und Übergangseffekte sparsam und sinnvoll einsetzen.
  • Verwendung von Flexbox oder Grid für Layouts.
  • Testen der Webseite in verschiedenen Browsern und Geräten.
  • Regelmäßige Validierung des CSS-Codes (z.B. mit dem W3C Validator).

3. JS - JavaScript erstellen

Was ist wichtig?

  • Externe JavaScript-Dateien verwenden.
  • Verwendung von semantischen und beschreibenden Variablennamen.
  • Funktionen sollten eine spezifische Aufgabe erfüllen (Single Responsibility Principle).
  • Verwendung von let und const anstelle von var.
  • Einheitliche Einrückung und Formatierung des Codes.
  • Kommentare zur Erklärung von komplexem Code verwenden.
  • Verwendung von Funktionen höherer Ordnung (z.B. map, filter, reduce).
  • Asynchrone Programmierung mit Promises oder async/await nutzen.
  • Fehlerbehandlung implementieren (z.B. try/catch).
  • Regelmäßige Validierung und Überprüfung des Codes (z.B. Linter wie ESLint).
  • Vermeidung von globalen Variablen zur Reduzierung von Konflikten.
  • Modularisierung des Codes (z.B. mit ES6-Modulen oder IIFE).
  • Verwendung von strict mode ("use strict";) zur Vermeidung von Fehlern.
  • Wiederverwendbare Komponenten erstellen (z.B. mit Frameworks wie React).
  • Testen des Codes (z.B. mit Unit-Tests oder Integrationstests).
